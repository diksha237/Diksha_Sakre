ChatGPT Efficiency Evaluation

Welcome! This project explores the efficiency of ChatGPT in generating accurate and timely responses across different domains. This website serves as a platform that documents the evaluation and showcases our findings. This project blends technical analysis and interactive website design to effectively present results and insights.

The combination of web development tools such as HTML, CSS and JavaScript along with technologies such as MongoDB, Node.js and ChatGPT, bring mondern software development and Artificial Intelligence research to create a website that is not only efficient but effective as well in presenting insights and data.

This project was in collaboration with Victoria Torkos.

Below we aim to evaluate ChatGPTs accuracy, response times and effectiveness in handling queries and provide additional information regarding the process of this assignment.

Research Approach

Database Management with MongoDB:

The foundation for data storage was MongoDB, which grouped questions and the metrics used to evalute them into categories. The question wording, anticipated responses, ChatGPT's response, response time, and if the response was accurate were all captured in each record.

Scalability was quaranteed by the use of MongoDB, which allowed the system to manage an expanding collection of domain-specific queries.

Server-Side Processing with Node.js:

To coordinate the communication between the database, the ChatGPT API, and real-time visualizations, a Node.js server was developed. The responsibilities included:

  • Fetching questions from MongoDB
  • Sending ChatGPT API queries
  • Recording answers and computing performance metrics

Data Visualization with Chart.js:

A clear visualization of ChatGPT's performance was made possible by implementing real-time charts using chart.js

The visualizations include graphs tracking correct vs. incorrect responses and visualizations of response time distributions across question domains.

ChatGPT Integration:

Real-time question processing was done using the ChatGPT API. The system used secure API key management to send ChatGPT randomly chosen questions, get the answers, and then store them in MongoDB.

The iterative approach ensured that every API call was documented, enabling accurate analysis of response quality.

Dataset Overview

Structure of Dataset:

Every entry included:

  • A set of questions intended to evaluate interpretative or factual knowledge.
  • 4 possible choices for an answer(A, B, C, D).
  • Ground truth data for accuracy comparison.
  • An area to save ChatGPT's response.
  • Metrics for evaluation stored in another collection:
    • Accuracy: Whether or not ChatGPT's response matched the ground truth.
    • Response Time: The amount of time it takes for ChatGPT to produce a response.

Domains of Evaluation:

The dataset was divided into three main categories, reflecting diverse fields of knowledge:

  • History: Questions covered significant historical events, figures, and periods to test factual recall and context understanding.
  • Social Science: Prompts dive into topics like psychology, sociology and economics to test the models understanding of interdisceplinary subjects.
  • Computer security: Focused on technical topics such as cryptography, cybersecurity frameworks, and network vulnerabilities.

Purpose of these evaluations

These tests were primarily meant to evaluate ChatGPT's capacity to produce correct, timely, contextually suitable answers in various fields. The goal was to understand how well it performs in handling factual, interpretive, and technical questions while identifying its strengths and limitations in different domains. By analyzing these factorsl the project aimed to provide a clear depiction of ChatGPTs capabilities and areas for improvement
